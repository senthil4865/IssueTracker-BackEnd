Events to listen

verifyUser event
This event ("verifyUser") has to be listened on the user's end to verify user authentication.
user will only be set as online user after verification of authentication token.
disconnect event (optional)
This event ("disconnect") has to be listened, when the user is disconnected to show proper message.
Auth Error (optional)
This event ("auth-error") has to be listened, when the user does not have valid auth token to do some tasks. So that it will show proper message to user.

Events to emit

set-user event
This event ("set-user") has to be emitted when a user comes online.
User can only be set as online only after verification of authentication token. Which you have pass here. The following data has to be emitted
authentication token
verifyUser event
This event ("verifyUser") has to be listened on the server's end to verify user authentication.
user will only be set as online user after verification of authentication token.
notify event
This event ("notify") has to be emitted to share the update message from one user to other users
The following data has to be emitted
                           {
                                userId: 'unique id of user',
                                message: 'message for user',
                                
                            }
disconnect event
This event ("disconnect") has to be emitted when a user disconnected or logged off from system
